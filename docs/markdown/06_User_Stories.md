# 1. Introduction

User stories describe system requirements from the user's perspective.

They focus on user needs rather than technical implementation details.

User stories help:

* understand user expectations,
* prioritize features,
* improve communication,
* support Agile development.

---

# 2. Purpose

The purpose of this document is to identify all major interactions between users and the system.

This document will later support:

* Use Case Diagrams
* Sprint Planning
* Product Backlog Creation
* System Design
* Testing Activities

---

# 3. User Roles

The following users have been identified.

1. Customer
2. Administrator
3. Inventory Manager
4. Business Owner
5. Delivery Personnel
6. AI Services

---

# 4. User Story Template

All user stories follow the format:

```text
As a <user>,
I want <goal>,
So that <benefit>.
```

# 5. Customer User Stories

---

## US-001 User Registration

As a customer,

I want to create an account,

So that I can purchase products and manage my orders.

Priority: High

---

## US-002 User Login

As a customer,

I want to login securely,

So that my personal information remains protected.

Priority: High

---

## US-003 View Profile

As a customer,

I want to view and update my profile,

So that my information remains accurate.

Priority: Medium

# 13. Story Prioritization

* Product Search
* Cart
* Orders
* Product Management

---

## Should Have

* Recommendations
* Forecasting

## Could Have

* Wishlist
* AI Chat Assistant

---

## Won't Have (Initial Version)

* Mobile Application
* International Shipping
* Multi-vendor Marketplace
